Introdução à Programação Orientada a Objetos (POO)
Introdução

A partir desta unidade, você entrará em uma nova fase do aprendizado em JavaScript: a Programação Orientada a Objetos (POO).
Até agora, você trabalhou principalmente com variáveis, arrays, funções e lógica procedural. Embora esse modelo funcione muito bem para programas pequenos, à medida que os sistemas crescem, torna-se necessário organizar melhor o código, facilitar a manutenção e reaproveitar funcionalidades.

É exatamente para isso que a Programação Orientada a Objetos existe.

O que é Programação Orientada a Objetos?

A POO é um paradigma de programação baseado no conceito de objetos.
Um objeto representa algo do mundo real ou lógico, como uma pessoa, um carro, um inimigo de um jogo ou um item de um sistema.

Cada objeto é composto por:

Propriedades → características do objeto

Métodos → ações que o objeto pode executar

Comparação com o que você já aprendeu

Antes da POO, você criava códigos assim:

let nome = "Carlos";
let idade = 20;

function apresentar() {
    console.log("Nome:", nome);
    console.log("Idade:", idade);
}


Com POO, esses dados e comportamentos ficam organizados em um único lugar.

O que é um Objeto?

Um objeto é uma estrutura que agrupa dados e comportamentos relacionados.

Exemplo simples de objeto:

let pessoa = {
    nome: "Carlos",
    idade: 20,
    apresentar: function () {
        console.log("Nome:", this.nome);
        console.log("Idade:", this.idade);
    }
};


Nesse exemplo:

nome e idade são propriedades

apresentar é um método

this se refere ao próprio objeto

Por que usar POO?

A Programação Orientada a Objetos permite:

Melhor organização do código

Facilidade de manutenção

Reaproveitamento de estruturas

Criação de sistemas mais complexos

Código mais legível e profissional

Essas características são essenciais em:

Jogos

Sistemas web

Aplicações grandes

Projetos em equipe
