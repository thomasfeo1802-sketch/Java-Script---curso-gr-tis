Transformando Lógica em Código: Estrutura de um Sistema

Na aula anterior, você aprendeu a pensar como um programador, planejando antes de escrever código.
Nesta aula, você vai aprender a transformar esse planejamento em código real, de forma organizada e profissional.

Aqui, o foco não é criar um programa pequeno, mas estruturar um sistema, mesmo que ele rode apenas no console.

1. Estrutura básica de um sistema

Todo sistema, por mais simples que seja, pode ser dividido em quatro partes principais:

Mensagem inicial

Dados do sistema

Funções (ações)

Execução do programa

Essa estrutura evita bagunça e deixa o código fácil de entender.

2. Mensagem inicial

Todo bom programa começa informando o usuário.

Exemplo de responsabilidades da mensagem inicial:

Mostrar o nome do sistema

Explicar rapidamente o que ele faz

Indicar que o programa foi iniciado corretamente

Isso ajuda tanto o usuário quanto o próprio programador durante testes.

3. Dados do sistema

Aqui ficam todas as informações importantes do programa, como:

Listas (arrays)

Objetos

Variáveis de controle

Estados do sistema

Exemplos:

Lista de produtos

Lista de usuários

Contadores

Configurações fixas

A regra é clara:

Dados devem ficar juntos e bem organizados.

4. Funções: o coração do sistema

Funções representam ações.

Cada função deve:

Ter uma única responsabilidade

Ter um nome claro

Resolver apenas um problema

Exemplos de ações:

Listar dados

Adicionar algo

Remover algo

Atualizar informações

Evite funções muito grandes.
Se uma função faz coisas demais, ela provavelmente deveria ser dividida.

5. Execução do programa

Depois de definir os dados e funções, o sistema começa a rodar.

Aqui você:

Chama funções

Simula ações

Testa comportamentos

Observa os resultados no console

Essa parte é onde o sistema “ganha vida”.

6. Ordem importa

Um erro comum de iniciantes é misturar tudo:

Função no meio do código

Variável depois da execução

Lógica espalhada

A ordem recomendada é:

Mensagem inicial

Dados

Funções

Execução

Isso deixa o código:

Mais legível

Mais profissional

Mais fácil de manter

7. Exemplo de pensamento correto

Antes de escrever:

O que o sistema faz?

Quais dados ele usa?

Quais ações existem?

Em que ordem tudo acontece?

Quem pensa assim escreve menos código errado.

8. Objetivo desta aula

Ao final desta aula, você deve conseguir:

Criar a estrutura base de um sistema

Organizar dados e funções corretamente

Evitar código confuso

Escrever programas mais claros e profissionais

Na próxima aula, você vai aprender a simular interações, criando sistemas que mudam de estado conforme o uso.

Conclusão:
Código bem escrito começa com organização.
Antes de pensar em “funcionar”, pense em fazer sentido.
